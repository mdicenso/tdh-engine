2 · Vincolo di licenza (decisivo)

Entrambe sono proprietarie. Dai Termini di Servizio di AirROI (verificati):

  • vietato «distribute, publish, display … the Data, in whole or substantial part» e «republish … for external use or circulation without prior express written permission»;
  • cache/conservazione dati API max 30 giorni + cancellazione a fine accesso;
  • licenza Commercial = uso interno (analisi, ricerca di mercato).
Conseguenze per il TDH:
  • ❌ i dati reali non si committano nel repo né si lasciano su un sito pubblico;
  • ✅ utilizzabili nel deploy privato (chiamata a runtime con chiave, cache ≤30gg, uso interno);
  • ⚠️ per vendere a clienti terzi un portale che mostra questi dati serve una licenza di redistribuzione / OEM da negoziare col fornitore (la licenza base non basta).

4 · Costo per coprire TUTTA ITALIA

AirROI si paga per mercato (aggregato), non per annuncio. Con ~100 mercati italiani a livello città/comune e le chiamate markets/metrics/* a $0,05-0,10:

  • snapshot nazionale una tantum: ~$5-10 (100 mercati × $0,05-0,10);
  • aggiornamento trimestrale: ~$20-40/anno (dimezzabile col piano «Preferred Partner»);
  • deposito minimo iniziale: $10.

AirDNA per la copertura nazionale via API è enterprise/su preventivo (ordini di grandezza superiori); gli abbonamenti SaaS (Research/Host) sono pensati per singoli host/gestori, non per estrarre dati nazionali via API.

Verdetto costo/beneficio: AirROI vince nettamente per la copertura nazionale a metriche reali. AirDNA conviene solo se servono profondità storica/benchmarking enterprise e c'è budget.

6 · Raccomandazione & prossimi passi

  1. Base gratuita: tenere Inside Airbnb (già integrata) come fondo pubblico/gratuito.
  2. Copertura nazionale reale: AirROI è la scelta di miglior rapporto costo/beneficio (~$5-10 per un ritratto di tutta Italia con occupazione/ricavi reali).
  3. AirDNA: valutare solo se servono storico profondo/benchmarking enterprise e c'è budget dedicato.
  4. Prima di attivare l'API reale: (a) registrazione + chiave API; (b) reader che sostituisce i dati d'esempio (cache ≤30gg, chiave nei secrets, dato non committato, solo deploy privato); (c) chiarire con AirROI/AirDNA la licenza di redistribuzione/OEM se il portale sarà venduto a terzi.

Il front-end è già stato prototipato nel TDH nella pagina «STR Italia (anteprima)» con dati sintetici sullo stesso schema, pronto a ricevere i dati reali.
